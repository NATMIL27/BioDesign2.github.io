"Diseño de un detector de convulsiones Tonicoclónicas orientado a niños con epilepsia"

¿Qué es un prototipo?

Un prototipo es un primer modelo que sirve como representación o simulación del producto final y que nos permite verificar el diseño y confirmar que cuenta con las características específicas planteadas.

SOLUCIÓN ÓPTIMA

Concepto de solución Nº3

El prototipo consiste  en un sistema arduino  montado en  un chaleco  de  material  ligero. El cual  tendrá  compartimentos para los  airbags que  protegerán cuello, extremidades  superiores y  cintura. El sistema  se acoplara cerca  del pecho, con acceso a la  batería recargable para su  reemplazo o recarga. Del módulo  arduino salen el  sensor de ritmo  cardiaco que puede  ser conectado  cerca del corazón por detrás  del chaleco, y el acelerómetro que solo debe estar  cerca del módulo de arduino. Un pulsador en la parte baja del chaleco por detrás, permitirá encender y apagar el sistema cuando el usuario lo decida. Todo el sistema estará oculto detrás de las costuras con acceso a la batería y a los airbags solamente.
La complejidad  del sistema radica en el  algoritmo de detección de convulsiones y de caídas, el cual determina el instante en el cual se activan los airbags. Todo el prototipo, salvo por la adición de airbags, debería ser de fácil fabricación, ya que consiste en  incluir el sistema  por entre  las costuras  del chaleco. Además  a nivel de hardware el sistema no es complejo, por  lo que  se  espera que  para el  diseño  final el sistema  sea más  simple que  el del  prototipo, quitando módulos  del arduino que no sean necesarios. Por último, los  airbags serán fabricados a medida por la empresa FESTO [67], los cuales serán fácilmente reemplazables después de una activación. 

Diseño y prototipado de los componentes de la solución

Una vez conseguida la solución óptima para la problemática planteada, procedemos a diseñar sus componentes. Dada las características de nuestro concepto de solución requerimos de dos diseños principales, el diseño electrónico y el diseño de software. El diseño electrónico será embebido dentro de las costuras y compartimientos del chaleco, el cual será modelado a partir de un chaleco estándar unisex para niños. El diseño de software estará cargado dentro del controlador principal (arduino) que operará de manera autónoma sin intervención del usuario.

El chaleco será simple y sin ningún diseño para efectos del prototipo. En la figura N°15 se observa el chaleco en su estado normal y en su estado activado. Elaboración Propia

A. Diseño y prototipado
del chaleco

El chaleco será confeccionado en tela sarga ya que otorgan mucha resistencia y durabilidad. Es un tejido elaborado con fibras entrecruzadas que le da flexibilidad a la prenda para que no se arrugue y, en muchos casos, que sea más resistente al agua que otras prendas. Para los compartimentos que cubren el sistema y los airbags se utilizará un tejido de microfibra de poliéster con un revestimiento impermeable especial, que es 100% impermeable, de esta manera se evita humedecer el sistema en caso el usuario se encuentra en un clima frío y húmedo, o lluvioso.

Es necesario aclarar que para propósitos comerciales el chaleco debería tener diseños llamativos para ser usados por niños y niñas, de esta forma nos aseguramos que sea una prenda que pase desapercibida en el día a día del paciente.

B. Diseño y prototipado electrónico

Como ya se había mencionado nuestro dispositivo está basado en el sistema Arduino, cuya función es la de controlar los sensores y actuadores, y procesar la data recolectada por estos. Presentamos entonces en la Figura N°17 el esquema de las conexiones necesarias para que el sistema funcione. Como se puede observar, las salidas de datos del sensor de ritmo cardíaco y del acelerómetro están conectados directamente a los pines de entrada analógica del Arduino Uno, y de la misma manera el botón de encendido y los airbags a los pines digitales. Así mismo, de los pines de voltaje y tierra del Arduino, sale la alimentación para todos los componentes

C. Diseño y prototipado de software

El software es la parte más importante del sistema, ya que será este el que se encargue de procesar las señales y activar los airbags en caso sea necesario. La programación de este software se debe hacer de tal forma que funcione de manera autónoma, así el usuario no tiene que hacer nada para operarlo.

1
El programa inicia declarando las librerías a utilizar, que son programas ya definidos que añaden nuevas funcionalidades a nuestro programa. Utilizaremos las librerías SPI.h, Wire.h, SparkFun_ADXL345.h y math.h, las dos primeras para la comunicación serial, la tercera para la operación del acelerómetro y la tercera para realizar operaciones matemáticas.
2
Después de declarar las variables necesarias para las operaciones creamos dos funciones, una para detectar un ritmo elevado mediante el sensor de ritmo cardiaco, y otra para detectar una caída mediante la lectura de las posiciones en el acelerómetro. La primera utiliza la medición de pulsos para determinar cuando el pulso del usuario sube a más de 100 latidos por minuto. Y la segunda utiliza el cambio de posiciones para transformarlo en aceleración y determinar si hay una aceleración específica que nos indique una caída.
3
Finalmente, el programa al detectar estas dos coincidencias enviará una señal digital a los airbags, los cuales se activarán y protegerán al usuario. Posterior a eso, y debido a que el sistema ya ha sido activado, este se apagará automáticamente para permitir el cambio de airbags.

// Librerías a utilizar
#include

#include

// Variables del SEN-11574
int PulsePin = 3; // Pin Analógico al que va conectado el Sensor SEN-11574
int Signal; // holds the incoming raw data. Signal value can range from 0-1024

int dR = 100; // Modificar de tal forma que detecte cuando hay un cambio brusco en el ritmo cardíaco.

// Creamos el objeto ADXL345
ADXL345 adxl = ADXL345();

// Variables del ADXL345
const int G = 76; // Constante de transformación
int x,y,z; // Variables de aceleración

// Variables para la bolsa de aire
int pinB = 3;

// Función para detectar el ritmo elevado
bool ritmoElevado(){
int S0 = analogRead(PulsePin);
int t0 = millis();
delay(10);
int S1 = analogRead(PulsePin);
int t1 = millis();
int dt = t1 - t0;

float dr = (S1-S0)/dt;

return dr > dR;
}

// Creamos la función para detectar una caída
bool caida(){
// Leemos los valores
adxl.readAccel(&x, &y, &z);

// Realizamos los cálculos
float G_x = x - 386;
float G_y = y - 388;
float G_z = z - 400;

// Transformamos los datos
float Gx = 1.0*G_x/G;
float Gy = 1.0*G_y/G;
float Gz = 1.0*G_z/G;

// Elevamos al cuadrado la gravedad de cada eje
float G2x = pow(Gx,2);
float G2y = pow(Gy,2);
float G2z = pow(Gz,2);

// Gravedad resultante
float G_resultante = sqrt(G2x + G2y + G2z);

return G_resultante > 2.3;
}

void setup() {
pinMode(pinB, OUTPUT);
Serial.begin(9600);
adxl.powerOn(); // Encendemos el ADXL345
adxl.setRangeSetting(16); // Definimos el rango (2, 4, 8 o 16)
}

void loop() {
if(ritmoElevado() && caida()){ // En caso de que haya caída y ritmo elevado se activa la bolsa de aire
digitalWrite(pinB, HIGH);
return 0; // Como ya hemos realizado la acción principal, el programa termina
}
delay(100);
}
